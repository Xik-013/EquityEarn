## Always

- Always read design-tokens.md and component-map.md before building any screen or component.
- Always use the canonical Logo/Full/Horizontal/Black component — never rebuild the logo from shapes and text. Key: 542dd8e54c0fd4fb3f202a024a0254289c8294a8 (📚 Equity Earn Library).
- Always use rescale() not resize() when scaling a Figma component instance with absolutely-positioned children (e.g. logo, icons).
- Always use the canonical Navigation/Nav bar/Desktop component on every post-login screen. Key: 332c90e7fd5e5d94929dc1627ee50a08eec39b61 (📚 Equity Earn Library).
- Always use the With Label component (key: 31498c87ad140feb48bc04bf2610be7ee1e81e03) for all form inputs — never a bare Input without a Label.
- Always set component properties via setProperties() — never edit locked text nodes directly inside instances.
- Always bind colours to variables using figma.variables.setBoundVariableForPaint() — never hardcode hex fills or strokes.
- Always apply published shadcn text styles to Figma text nodes via importStyleByKeyAsync — never set raw font properties.
- Always use Text-2xl/Bold for all screen titles, card titles, and modal titles.
- Always reuse an existing component before building a new one.

## Never

- Never use rounded-[14px] on inputs. Never use rounded-md on buttons. See radius rules in design-tokens.md.
- Never hardcode #737373, #71717b, #e4e4e7, #ddd or any token colour — always bind to a variable.
- Never use Figma MCP temp asset URLs in production code — export assets to /public/images/.
- Never rebuild the logo manually (circle + text). Always import the library component.
- Never create a nav bar as a plain FRAME on post-login screens — always use the canonical instance.
- Never edit files in components/ui/ — those are auto-generated by shadcn.
- Never add a dependency without asking first.
- Never refactor code unrelated to the task.
- Never edit Figma library files without explicit confirmation for that specific edit.

## Definition of done

- Matches the Figma screenshot, not just close
- Uses tokens from design-tokens.md and components from component-map.md — nothing invented
- All colour fills/strokes bound to variables — zero hardcoded hex
- pnpm check passes (lint, types, tests)

## Handoff workflow

1. Designer frames screen in Figma using library components
2. Share node URL → Claude reads via Figma MCP
3. Claude reads design-tokens.md + component-map.md
4. Claude generates React component following these conventions
5. Dev pastes into correct app/ directory
6. Export Figma assets → /public/images/
7. Push to GitHub → Vercel creates preview URL
8. Designer reviews preview → approves or flags issues
9. Merge to main → auto-deploys
